# 岩土工程 AI 辅助写作系统

> 从 PDF 勘察报告到结构化参数，再到标准报告正文——规则提取 + 模板生成，不是开放式文本生成。

**当前版本：v3.0** | 2026-03-30

---

## 项目简介

本系统面向岩土工程勘察报告场景，提供一套完整的 AI 辅助写作工作流：

```
PDF 勘察报告 → 文本/表格提取 → A~F 模块参数抽取 → 参数锁定表 → 动态章节生成 → Word 导出
```

核心设计原则：**所有参数从知识库和提取引擎读取，AI 不自行编造数值。**

## 系统能力

| 能力 | 说明 |
|------|------|
| PDF 读取 | OpenDataLoader（推荐）+ pdfplumber 双引擎，文本和表格高精度提取 |
| 数据提取 | 6 大模块自动抽取（A 项目信息 / D 试验参数 / E 地下水 / F 抗震） |
| 参数锁定表 | 自动生成 Markdown 格式，人工可核对，是写作与提取之间的可审查中间态 |
| 模板引擎 | 150 条标准表达 + 105 条结论模板 + 25 个专用场景函数 |
| 动态章节 | 4 个报告章节（概况/抗震/地基/结论）基于提取数据动态生成，缺失时安全降级 |
| Word 导出 | Markdown 转 Word，支持自定义企业模板 |

## 目录结构

```
geo_report_workflow.py      # 主工作流（入口文件）
opendataloader_pdf_reader.py # PDF 读取引擎
geo_writer.py               # 模板引擎入口
geo_writer_templates.py     # 150 条标准表达模板（EXP-001~150）
geo_writer_conclusions.py   # 105 条结论模板（CON-001~105）
geo_writer_specialized.py   # 25 个专用场景函数
geo_word_exporter.py        # Word 导出器
geo_logger.py               # 统一日志模块
```

## 运行环境

- Python 3.9+
- Windows / macOS / Linux

### 依赖安装

```bash
# 必选
pip install pdfplumber python-docx

# 推荐（PDF 提取首选引擎，需要 Java 11+）
pip install -U opendataloader-pdf

# 系统要求（仅 opendataloader-pdf）
# Java 11+ 必须在系统 PATH 中
```

> 如果不安装 `opendataloader-pdf`，系统会自动降级使用 pdfplumber，但表格提取精度会降低。

## 使用方式

### 演示模式

不带参数运行，使用内置演示数据验证系统：

```bash
py geo_report_workflow.py
```

输出：
- `<项目名>_项目参数锁定表.md` — 参数中间文件
- `<项目名>_AI生成报告/` — 包含 4 个 Markdown 章节文件
- `<项目名>_完整报告.docx` — 合并后的 Word 文档

### 真实项目模式

传入项目文件夹路径，系统自动扫描 PDF 并执行完整流程：

```bash
py geo_report_workflow.py "H:/项目/2026-KC-XTFY21邢台双创智慧谷项目1#商务楼及地下车库"
```

系统会自动：
1. 扫描文件夹中的 PDF 文件
2. 提取文本和表格数据
3. 按正则规则抽取 A~F 模块参数
4. 生成参数锁定表（可人工核对修改）
5. 基于参数动态生成报告章节
6. 导出 Word 文档

### 模板引擎单独使用

```python
from geo_writer import *

# 调用专用函数
print(承载力建议('三', '细砂', '180'))
print(桩基持力层建议('十', '中砂', '3000', '六', '粉质黏土', '900'))
print(场地稳定性结论())
print(液化判别结论('不液化', '0'))
print(腐蚀性结论('微腐蚀性'))
print(勘察等级结论('二', '二', '二', '甲'))
```

## 当前能力边界

- **D/E/F 模块**提取较成熟（基于 9 份真实报告验证，准确率 90%+）
- **A 模块**（项目基础信息）仍需增强，当前提取能力有限
- 适用于标准化岩土工程勘察报告（建筑类、管线类）
- 极端格式 PDF 和特殊项目类型仍需人工复核
- 参数锁定表中标记为"待确认"的字段需人工补填
